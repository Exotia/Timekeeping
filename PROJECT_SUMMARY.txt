# Project Summary: Command-Line Timekeeping Application

## 1. Project Overview

**Purpose:** A robust, visually enhanced command-line Python tool for tracking work hours, calculating time balances against a daily target (1.8h), and generating detailed, color-coded reports.

**Core Files:**
- `time` / `time.py`: The executable Python script (Main application).
- `timesheet.csv`: The data file where all time entries and comments are stored.

---

## 2. How to Use

### 2.1. Initial Setup
The script is designed to be run directly. Make it executable first:
`chmod +x time`

### 2.2. Adding a New Time Entry
The script is run with an optional command-line argument to specify the date.

- **For Today:** `./time`
- **For a Different Day:** `./time [offset]`, where `[offset]` is a number.
  - Example for yesterday: `./time -1`
  - Example for tomorrow: `./time 1` (Note: The tool strictly prevents bookings on weekends).

### 2.3. Bulk Entries
Record multiple special days at once:
- **Bulk Gleitzeit:** `./time --bulk-gleitzeit`
- **Bulk Urlaub:** `./time --bulk-urlaub`
Follow the prompts to enter a start and end date (or offsets). The tool automatically skips weekends and days with existing entries during bulk operations.

### 2.4. Viewing the Timesheet
To view the formatted reports, use the `--view` flag:
`./time --view`

---

## 3. Core Features in Detail

### 3.1. Visual & Interactive UI
- **Color Coding:** 
    - **Green:** Successful actions, positive balances, days with bookings (Day column), and "URLAUB" banners.
    - **Deeper Red:** Warnings, errors, negative balances, and "NEEDS ATTENTION" flags.
    - **Cyan:** Interactive questions.
    - **Yellow:** Weekend banners.
    - **Dim:** Placeholders (`_`), examples, and existing booking info.
- **Global Exit:** Type **'b'** at any prompt to exit the application immediately.
- **Full Date Display:** Dates are presented in a readable format in success messages and headers (e.g., `2. February 2026`). 
- **Consolidated Day Column:** In the reporting table, the Day column shows `Abbr DD` (e.g., `Mon 02`) and is only displayed for the first entry of that day.

### 3.2. Entry Types
- **Regular Work:** Enter start/stop times (e.g., `800`, `1730`). Allows optional comments and project selection from a configurable list.
- **"Gleitzeit-Tag" (Flex Day):** Enter **'0'** as start time. Logs zero hours but applies the daily 1.8h debt. Only one Gleitzeit-Tag is allowed per day.
- **"Urlaub" (Vacation):** Enter **'u'** as start time. Logs zero hours and **skips** the 1.8h daily target deduction. 

### 3.3. Conflict Resolution & Validation
If you try to add a conflicting entry (overlap, adding work to a special day, etc.), a menu appears:
1.  **Stop and cancel:** Aborts the operation.
2.  **Overwrite all entries for this day:** Clears the day and starts fresh.
3.  **Move this booking to another day:** Prompts for an offset and restarts validation for the new date.
4.  **Re-enter input:** Allows correcting the time or type for the current day.

*Note: Special days (Gleitzeit/Urlaub) are displayed by name in the "Existing bookings" list for clarity.*

### 3.4. Automatic Calculations
- **Break Rules:** > 3 hours: -18 minutes; > 6 hours: -48 minutes total.
- **Workday Target:** Hardcoded at **1.8 hours** per workday (Monday-Friday).
- **Past-Only Scope:** "Total Netto Time", "Project Summary", and "General Balance" only include data from the **past and today**. Future bookings are visible in tables but excluded from statistics.

### 3.5. Configuration
The `time` script contains a configuration section at the top:
- `START_DATE_OVERRIDE`: Define exactly when your balance tracking should begin (e.g., `"2026-01-01"`).
- `INITIAL_BALANCE_HOURS`: Manually set a starting balance (e.g., `-15.9`) from previous systems.
- `PROJECTS`: A list of project names to choose from during time entry.

### 3.6. Reporting Tool (`--view`)
- **Consolidated Tables:** Cleaner view for days with multiple bookings.
- **Statistics:** Includes monthly project breakdowns (excluding Urlaub) and running balances.
- **Missing Entries:** Past weekdays with no entries are flagged with a centered red `!! NEEDS ATTENTION !!`.
- **Special Row Banners:** Weekend (Yellow) and Urlaub (Green) banners are centered across the entire row.
